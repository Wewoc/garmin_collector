How a working session between a non-programmer and an AI coding partner actually flows — from first impulse to committed change.

User decides

Every decision — scope, architecture, option selection — rests with the user. The AI does not proceed without explicit instruction.

Assess first

Before any code is written, the user requests evaluation. "Concept first — nothing yet." A deliberate control structure, not a workaround.

AI implements

Complete files only — never snippets. One change per step. Internal check first, then deliver. Working code is treated as sacred.

Multi-AI review

Major decisions cross-checked with Claude, Gemini, ChatGPT, Copilot independently. Intersection of findings = true signal.

Correction / brake

User corrects AI context errors immediately. AI brakes scope drift explicitly — "OS task, not application." Runs both directions.

Handoff

Every session ends with NOTES, CHANGELOG, REFERENCE update, and a versioned START_PROMPT. Context survives without memory.
